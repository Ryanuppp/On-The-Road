# 旅行攻略生成

## 总体流程

```
1. 搜索小红书 → 获取景点/美食/住宿推荐
2. 高德地图地理编码 → 获取所有景点精确坐标
3. 高德地图路线规划 → 计算各段距离/时间/费用
4. 撰写 Markdown 攻略 → 严格按模板格式
5. 更新 plans.json → 注册新攻略
6. 本地验证 → 启动 HTTP server + 浏览器截图确认渲染
```

## 一、信息收集（小红书）

使用 `xiaohongshu-search` skill 搜索，建议搜索的关键词组合：

| 场景 | 关键词示例 |
|------|-----------|
| 综合攻略 | `"{城市}+天数+攻略"` |
| 美食 | `"{城市} 必吃 美食"` |
| 亲子 | `"{城市} 亲子 带娃"` |
| 带老人 | `"带父母 {城市}"` |
| 季节 | `"{月份} {城市} 穿衣 天气"` |

**要点**：
- 同时搜 3–5 组关键词，交叉验证信息
- 优先采信高赞笔记（>100 赞），但低赞笔记可能有独到本地人视角
- 在攻略中标注信息来源：`📖 **小红书来源**：[标题 · N赞](链接)`
- 笔记文字内容很短但图片多时，需下载图片读取（详见 xiaohongshu-search skill）
- **⚠️ 必须过滤广告帖**：只采信个人用户的真实体验帖，排除旅行社/地接/包车等商业账号。详细过滤规则见 `xiaohongshu-search` skill 的「广告过滤」章节

## 二、路线规划（高德地图 / Google Maps）

### 高德地图（国内目的地）

使用 `user-高德地图Amap` MCP server。

#### 地理编码（关键步骤）

```
tool: maps_geo
args: {"address": "完整地址", "city": "城市名"}
```

**⚠️ 地理编码陷阱（必读）**：
- 用简称（如"狮子林"）会匹配到其他省份的同名地点
- **必须写完整结构化地址**：`{城市}市{区}区{路}{号}{景点名}`
- 示例：`苏州市姑苏区园林路23号狮子林` ✅ vs `狮子林` ❌
- 每次编码后**验证坐标**：经纬度应在目标城市范围内

#### 路线查询

| 工具 | 用途 | 关键参数 |
|------|------|----------|
| `maps_direction_driving` | 打车路线 | `origin`, `destination`（经纬度格式：`lng,lat`） |
| `maps_direction_walking` | 步行路线 | 同上 |
| `maps_direction_transit_integrated` | 地铁/公交 | 同上 + `city` + `cityd` |

- origin/destination 格式：`"经度,纬度"`（注意：高德是 `lng,lat`，与 Google Maps 的 `lat,lng` 相反）
- 多段路线可并行查询，一次性获取全天数据

#### 高德导航链接格式

```
https://uri.amap.com/navigation?from={lng},{lat},{起点名}&to={lng},{lat},{终点名}&mode={car|walk|bus}
```

地点标记链接：
```
https://uri.amap.com/marker?position={lng},{lat}&name={地点名}
```

### Google Maps（境外目的地）

参见 `google-maps-route-planning` skill。主要区别：
- 坐标格式 `lat,lng`（与高德相反）
- 步行链接需追加 `data=!4m2!4m1!3e2`

## 二-B、丰富已有景点详情（小红书深度搜索）

当用户要求补充景点可玩内容时，针对每个景点做多轮细分搜索。

### 搜索关键词策略

| 目标 | 关键词模板 | 示例 |
|------|-----------|------|
| 景点玩法 | `"{城市} {景点名} 攻略 拍照"` | `"海参崴 托卡列夫灯塔 攻略 拍照"` |
| Citywalk 路线 | `"{城市} {景点A} {景点B} {景点C} 攻略"` | `"海参崴 鹰巢山 缆车 潜艇博物馆 火车站 攻略"` |
| 徒步/野外 | `"{城市} {景点} 徒步 {相邻景点}"` | `"海参崴 托比津纳海角 徒步 玻璃海滩"` |
| 购物/商场 | `"{城市} {商场名} 逛"` | `"海参崴 步行街 FEFU 远东联邦大学 逛"` |
| 涂鸦/拍照 | `"{城市} 涂鸦墙 古建筑 步行街 拍照打卡"` | |
| 天气穿衣 | `"{城市} {月份} 天气 穿衣 温度 衣服"` | `"海参崴 五月 天气 穿衣 温度 衣服"` |

**技巧**：
- 每次搜 `limit: 5`，多组关键词**并行搜索**（一次 message 发多个 CallMcpTool）
- 交叉验证多位博主的信息，尤其是价格、开放时间、路况等时效性强的内容
- 提取的有用信息：最佳时间段、拍照机位、注意事项（风/信号/鞋子）、厕所位置、实际体验感受、避坑提醒

### 景点详情卡片格式

每个景点独立一个小节，格式如下：

```markdown
### N. 景点名

> EN: English Name ｜ RU: Русское название ｜ [中文介绍](链接)

**背景**：1-2 句话概述历史/地位。

**详细玩法**：
- 🚶 交通/到达方式
- 🌅 最佳游玩时间
- 📸 拍照机位/技巧
- ⚠️ 注意事项（信号/风/鞋子/安全）
- 💡 实用小贴士
- 🎫 门票/费用

> 📖 [笔记标题 · N赞](链接)
```

## 二-C、出发前必备事项（国际目的地）

当目的地为境外城市时，在攻略开头添加「〇、出发前必备事项」章节，涵盖以下子项：

```markdown
## 〇、出发前必备事项（{出发城市}出发）

### 📋 证件与签证
| 项目 | 详情 |
| --- | --- |
| **护照** | 有效期要求 |
| **签证** | 申请方式、费用、出签时间、所需材料 |

### ✈️ 机票
| 项目 | 详情 |
| --- | --- |
| **航线** | 出发城市 → 目的地，直飞/中转，飞行时间 |
| **航空公司** | 常飞航司 |
| **参考价格** | 往返含税参考区间 |

### 💰 换汇与支付
| 项目 | 详情 |
| --- | --- |
| **国内提前换** | 换汇渠道、面额建议 |
| **当地换汇** | 推荐换汇点、汇率参考 |
| **支付方式** | 现金/刷卡/移动支付 |

### 📱 通讯与必装 App
| 项目 | 详情 |
| --- | --- |
| **电话卡** | 推荐运营商、购买渠道、流量 |
| **必装App** | 打车/翻译/地图/外卖 等 |
| **离线资料** | 离线地图/翻译包（无信号区域必备） |

### 🔌 随身物品清单
| 类别 | 物品 |
| --- | --- |
| **证件** | 护照、签证打印件、身份证 |
| **电子设备** | 充电宝、转换插头（标注插头标准）、数据线 |
| **穿着** | 根据目的地气温和活动类型建议 |
| **日用品** | 酒店不提供的物品提醒 |
| **药品** | 常用药物 |
```

**搜索关键词**：`"{出发城市}出发 {目的地} 签证 准备 必备"` + `"{目的地} 旅游备忘录"`

**信息来源**：优先采信高赞的行前准备类笔记（通常 >100 赞），注意提取"血泪教训"类经验（如落地没钱、信号问题等）作为醒目提醒。

## 三、Markdown 攻略格式

严格参照 `plans/Vladivostok.md` 的章节结构。必需章节：

```markdown
# {emoji} {城市}N日旅行计划

> **时间** / **路线** / **节奏** / **旅行主线** / **阅读顺序**

## 一、路线总览          ← 全程表格：Day / 主题 / 交通 / 重点
## 二、住宿安排          ← 推荐区域 + 理由
## 三、N日行程总表       ← 每日 timeline 表格
## 四、交通费用总览       ← 打车/门票费用表 + 省钱提示
## 五、每日细节           ← 核心！逐小时行程 + 交通详情表 + 导航链接表
### Day N ｜ 日期 ｜ 主题
## 六、实用信息           ← 出行/天气/支付/预约等
## 七、全程路线汇总       ← 每日全程里程 + 所有导航链接
## 八、坐标速查           ← 所有景点坐标 + 地图链接
## 九、景点介绍           ← 每个景点的背景 + 玩点 + 适宜度
## 十、美食推荐汇总       ← 必吃清单 + 推荐餐厅表
## 十一、特别提醒         ← 按出行人群分类的注意事项
## 十二、可选活动         ← 备选方案 + 小红书来源汇总
```

### 每日细节的固定子结构

```markdown
### Day N ｜ 日期 周几 ｜ 主题

> 交通/节奏提示 blockquote

**今日重点**：一句话概括

**行程节点**：
1. **HH:MM** 景点名 ⭐ 标签（停留 N 分钟）
   - 🎫 门票信息
   - 💡 小贴士
   - 📖 **小红书来源**：[标题 · N赞](链接)

#### 📍 Day N 交通详情（高德地图实测）

| 路段 | 方式 | 距离 | 时间 | 预估费用 |
| --- | --- | --- | --- | --- |

**逐段导航链接（高德地图）**：

| # | 路段 | 模式 | 导航 |
| --- | --- | --- | --- |

> ⚠️ **注意**：当日特殊提醒
```

### 每日餐厅推荐格式

每餐提供 **2–3 个临近餐厅选项**，以 `#### 🍽️` 开头的 h4 标题 + 表格呈现。`index.html` 会自动将其渲染为卡片样式。

- 国内目的地：每个餐厅附 **大众点评** + **高德地图** 链接
- 国外目的地：每个餐厅附 **大众点评** + **谷歌地图** 链接
- 餐厅表格放在**行程节点**列表结束后、**📍 Day N 交通详情**之前
- 表头固定为：`| 餐厅 | 推荐菜 | 人均 | 链接 |`
- 链接格式：`[大众点评](url) [高德地图](url)` 或 `[大众点评](url) [谷歌地图](url)`

链接模板：
- 大众点评（国内）：`https://www.dianping.com/search/keyword/{cityId}/0_{餐厅名}`
- 大众点评（国外）：`https://www.dianping.com/search/keyword/0/0_{餐厅名}{城市中文名}`
- 高德地图：`https://uri.amap.com/search?keyword={餐厅名}&city={城市名}`
- 谷歌地图：`https://www.google.com/maps/search/?api=1&query={餐厅名}+{城市英文名}`

### 格式规范

- checkbox 行用 `- [ ]` 开头，`index.html` 会渲染为可勾选进度条
- emoji 统一用法：🚕打车 🚶步行 🚇地铁 🚄高铁 🏛️博物馆 ⛩️寺庙 🏮古街 🗿景区 🎫门票 💡提示 📖来源 ⚠️注意
- 表格对齐用 `---`，不需手动对齐空格
- 导航链接统一用 `[导航](URL)` 或 `[打开](URL)`

## 五、特殊人群注意事项（经验总结）

### 亲子出行（含幼儿）

| 维度 | 要点 |
|------|------|
| 节奏 | 每天最多 3–4 个景点，景点间打车衔接 |
| 午睡 | 中午安排回酒店午休 1–1.5 小时 |
| 推车 | 建议带轻便口袋推车，古城/园林部分路段不好推但比全程抱强 |
| 安全 | 假山/台阶/水边等场景特别提醒 |
| 兴趣点 | 标注每个景点的儿童友好度（⭐评分），优先安排互动性强的 |
| 饮食 | 备注口味是否适合儿童，建议自备零食 |

### 带老人出行

| 维度 | 要点 |
|------|------|
| 体力 | 景点间尽量打车，减少步行量 |
| 优惠 | 标注老年人门票政策（60岁/65岁/70岁各档优惠） |
| 节奏 | 每个景点留充足时间，不赶路 |
| 拍照 | 提前选好拍照机位，老人会很开心 |

### 打车费用估算

```
国内参考公式：
预估费用 = 起步价 + max(0, 距离km - 起步里程) × 每公里单价

常见城市起步价：
- 苏州：12元/3km，之后 2.3元/km
- 查询当地出租车费率后写入攻略
```

## 六、常见问题

| 问题 | 解决方案 |
|------|----------|
| 高德地理编码返回错误坐标 | 使用完整结构化地址，编码后验证坐标范围 |
| 小红书笔记内容为空/很短 | 信息在图片中，需下载 webp → 转 png → Read 读取 |
| Chrome DevTools 的 `navigate_page` 改 hash 失败 | 用 `evaluate_script` 执行 JS 修改 `window.location.hash` |
| 门票价格不确定 | 标注"参考价"并注明信息来源日期 |
| 多个同名景点 | 地理编码时加城市+区+完整路名 |
